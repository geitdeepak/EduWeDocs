    üõ†Ô∏è AWS Labs Troubleshooting Guide¶

    Welcome to the AWS Troubleshooting Document for EduWe learners. This guide helps you troubleshoot common AWS issues encountered during hands-on labs, whether you're using the AWS Free Tier, student sandbox, or personal account.

    üîê 1. Cannot Log in to AWS Console¶

    Issue: Login fails or shows invalid credentials.
    Possible Causes: - Typo in account ID, IAM username, or password. - Using root account where IAM login is needed (or vice versa). - MFA not configured or expired.

    Solution: - Use the IAM sign-in URL:
    https://<account-alias>.signin.aws.amazon.com/console - Double-check IAM user credentials. - Learn about IAM login:
    üëâ https://docs.aws.amazon.com/IAM/latest/UserGuide/console.html

    üì¶ 2. "Service Not Available in This Region"¶

    Issue: Selected service or AMI not available in the selected region.
    Cause: AWS services are region-specific.

    Solution: - Switch to supported regions like us-east-1 (N. Virginia) or ap-south-1 (Mumbai). - Check service availability:
    üëâ https://aws.amazon.com/about-aws/global-infrastructure/regional-product-services/

    üí≥ 3. AWS Free Tier Exhausted or Not Applied¶

    Issue: You get charged or blocked from creating resources.
    Causes: - Free tier limits exceeded. - Non-eligible instance or service selected.

    Solution: - Use free-tier eligible services like t2.micro. - Monitor free tier usage:
    üëâ https://console.aws.amazon.com/billing/home#/freetier - Learn more:
    üëâ https://aws.amazon.com/free

    üñ•Ô∏è 4. EC2 Instance Won‚Äôt Start or Stops Unexpectedly¶

    Issue: Instance fails to launch or stops on its own.
    Causes: - Quota limits, unsupported instance type, or key pair issues.

    Solution: - Choose a free-tier eligible AMI like Amazon Linux 2. - Verify key pair during launch:
    üëâ https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-key-pairs.html - Check service quotas:
    üëâ https://console.aws.amazon.com/servicequotas/home

    üîë 5. SSH or RDP Connection Fails¶

    Issue: You can‚Äôt connect to EC2 instance.
    Causes: - Incorrect security group settings. - Wrong or missing key pair. - No public IP assigned.

    Solution: - Add inbound rule for port 22 (SSH) or 3389 (RDP):
    üëâ https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/authorizing-access-to-an-instance.html - Use correct .pem file and run:
    chmod 400 your-key.pem - Ensure auto-assign public IP is enabled during instance setup.

    üìÅ 6. Can't Upload Files to S3 Bucket¶

    Issue: Upload fails or is denied.
    Causes: - Permissions missing, public access blocked, or large file.

    Solution: - Verify IAM policy allows s3:PutObject:
    üëâ https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies.html - Learn about S3 Block Public Access:
    üëâ https://docs.aws.amazon.com/AmazonS3/latest/userguide/access-control-block-public-access.html - Use AWS CLI for large file uploads:
    üëâ https://docs.aws.amazon.com/cli/latest/reference/s3/

    üßæ 7. Access Denied While Creating Resources¶

    Issue: AccessDenied or UnauthorizedOperation errors.
    Causes: - IAM user lacks permissions.

    Solution: - Admin should assign policies such as AmazonEC2FullAccess or PowerUserAccess:
    üëâ https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_job-functions.html - View permissions here:
    üëâ https://console.aws.amazon.com/iam/home#/users

    üóÉÔ∏è 8. Missing Key Pair or Key Not Downloaded¶

    Issue: Lost .pem file after creation.
    Solution: - You cannot recover the key later. - Create a new key pair and instance:
    üëâ https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-key-pairs.html - Alternatively, migrate the volume to a new instance.

    üì° 9. VPC/Subnet Not Found or Network Issues¶

    Issue: Can't launch resources due to missing VPC/subnet.
    Cause: VPC deleted or not configured correctly.

    Solution: - Use default VPC:
    üëâ https://docs.aws.amazon.com/vpc/latest/userguide/default-vpc.html - Create new VPC from console:
    üëâ https://docs.aws.amazon.com/vpc/latest/userguide/what-is-amazon-vpc.html

    • Use CloudShell for CLI-based tasks:
      üëâ https://console.aws.amazon.com/cloudshell
    • Monitor cost and usage:
      üëâ https://console.aws.amazon.com/billing/home
